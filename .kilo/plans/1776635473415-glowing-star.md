# Plan de Implementación: Firmware Modular ESP32 para Puente MAVLink → AWS IoT

## Contexto
Firmware para ESP32 que actúa como puente entre sistema de navegación MAVLink, sensores locales (2 analógicos: turbidez/pH, 1 temperatura: MAX31865) y AWS IoT Core usando módem A7670SA. Arquitectura modular con lógica no bloqueante.

## 2. Especificaciones de Hardware y Pinout

### UART
- **Serial1 (TX1=GPIO17, RX1=GPIO16)** → A7670SA (115200 baud)
- **Serial2 (TX2=GPIO2, RX2=GPIO4)** → Flight Controller MAVLink (57600 baud)

### SPI (MAX31865 - RTD/termopar)
- **SCK=GPIO18, MISO=GPIO19, MOSI=GPIO23, CS=GPIO5**

### Sensores Analógicos
- **Turbidez:** GPIO34 (ADC1_CH6)
- **pH:** GPIO35 (ADC1_CH7)

### LEDs de Estado
- **GPIO4** (o alternativo disponible): LED conectividad GSM
- **GPIO2** (alternativo): LED actividad MAVLink

## 3. Responsabilidades de Clases

### 3.1 GsmManager
**Responsabilidades:**
- Inicializar módem A7670SA mediante TinyGSM
- Verificar/crear sistema de archivos del módem
- Escribir certificados AWS en FS (primera ejecución)
- Establecer conexión GPRS robusta con reintentos agresivos
- Inicializar cliente MQTT (PubSubClient) y conectar a AWS IoT Core
- Suscribirse a tópicos de comando (ej. `usv/command`)
- Mantener loop MQTT no bloqueante
- Enviar payload JSON a tópico `usv/telemetry`
- Gestionar reconexión en segundo plano sin bloquear otras tareas

**Estado interno:**
- `enum class State { INIT, WAIT_CARD, GPRS_ATTACH, GPRS_ACTIVE, MQTT_CONNECT, READY, ERROR }`
- Contador de fallos consecutivos para resetear módem (límite: 3)
- Marcas de tiempo para reintentos: `lastGprsAttempt`, `lastMqttAttempt`
- `bool certificatesWritten` para skip en boots subsiguientes

**Métodos principales:**
- `void begin()`: Configura Serial1, inicializa TinyGSM
- `void update()`: Lógica principal no bloqueante, debe llamarse en cada loop
- `bool sendTelemetry(JsonObject& data)`: Envía payload a AWS
- `bool isConnected()`: Estado MQTT conectado
- `void handleIncoming()`: Process MQTT incoming messages

**Lógica de certificados (primera ejecución):**
```cpp
// Pseudocódigo
if (!fileExists("/cert/ca.pem")) {
  writeFile("/cert/ca.pem", AWS_ROOT_CA);
  writeFile("/cert/client.crt", CLIENT_CERT);
  writeFile("/cert/client.key", CLIENT_KEY);
  // Verificar con AT+FS LS=/cert/
}
```

### 3.2 MavlinkHandler
**Responsabilidades:**
- Configurar Serial2 a 57600 baud para MAVLink
- Parsear mensajes MAVLink v2 no bloqueante
- Extraer y almacenar último HEARTBEAT (sistema armado, modo vuelo)
- Extraer y almacenar último GLOBAL_POSITION_INT (lat, lon, alt, hdg)
- Mantener secuencia MAVLink
- No bloquear: leer bufferSerial disponibles, procesar solo bytes completos

**Estado interno:**
- `mavlink_message_t lastMsg;`
- `uint32_t lastHeartbeatMs;`
- `mavlink_global_position_int_t lastPos;`
- `bool hasValidPosition;`

**Métodos principales:**
- `void begin()`: Inicia Serial2, configura pins
- `void update()`: Lee Serial2.available(), parsea mensajes con mavlink_parse_char
- `bool getHeartbeat(mavlink_heartbeat_t& hb)`: Copia último heartbeat
- `bool getPosition(mavlink_global_position_int_t& pos)`: Copia última posición
- `bool isConnected()`:心跳超时判定 (ej. heartbeat perdido > 5s)

### 3.3 SensorManager
**Responsabilidades:**
- Inicializar ADC para sensores analógicos (attenuation ADC_11db para rango 0-3.3V)
- Inicializar SPI y MAX31865 para temperatura
- Leer sensores cada intervalo definido (ej. 100ms)
- Aplicar filtros: promedio móvil N muestras (N=4) para analógicos
- Convertir lecturas:
  - Turbidez: voltaje → NTU (fórmula calibrada)
  - pH: voltaje → pH (fórmula calibrada)
  - Temperatura: MAX31865 Reads → °C (resistencia PT100/termopar)
- Detectar errores de sensores (valores fuera de rango, desconectado)

**Estado interno:**
- `float turbidityRaw[4], phRaw[4];` (buffers circulares)
- `int bufferIdx;`
- `float turbidityFiltered, phFiltered;`
- `float temperature;`
- `uint32_t lastReadMs;`

**Métodos principales:**
- `void begin()`: Configura ADC, SPI, MAX31865
- `void update()`: Lee y filtra sensores si pasó intervalo (no bloqueante)
- `float getTurbidity()`, `float getpH()`, `float getTemperature()`

## 4. Flujo Principal (main.ino)

```cpp
#include "libs/GsmManager.h"
#include "libs/MavlinkHandler.h"
#include "libs/SensorManager.h"
#include "certs/aws_root_ca.h"
#include "certs/client_cert.h"
#include "certs/client_key.h"

GsmManager gsm(Serial1);
MavlinkHandler mavlink(Serial2);
SensorManager sensors;

const unsigned long TELEMETRY_INTERVAL = 1000; // 1 segundo
unsigned long lastTelemetryMs = 0;

void setup() {
  Serial.begin(115200);
  delay(100); // Solo en setup permitido
  
  sensors.begin();
  mavlink.begin();
  gsm.begin(AWS_ENDPOINT, AWS_ROOT_CA, CLIENT_CERT, CLIENT_KEY);
}

void loop() {
  unsigned long now = millis();
  
  // 1. Actualizar módulos (no bloqueante)
  sensors.update();
  mavlink.update();
  gsm.update(); // Incluye reconexión GPRS/MQTT en background
  
  // 2. Cada TELEMETRY_INTERVAL: construir y enviar payload
  if (now - lastTelemetryMs >= TELEMETRY_INTERVAL && gsm.isConnected()) {
    lastTelemetryMs = now;
    
    DynamicJsonDocument doc(512);
    JsonObject payload = doc.to<JsonObject>();
    
    // Sensores
    payload["turbidity"] = sensors.getTurbidity();
    payload["ph"] = sensors.getpH();
    payload["temperature"] = sensors.getTemperature();
    
    // MAVLink
    mavlink_heartbeat_t hb;
    if (mavlink.getHeartbeat(hb)) {
      JsonObject hbObj = payload.createNestedObject("heartbeat");
      hbObj["system_status"] = hb.system_status;
      hbObj["base_mode"] = hb.base_mode;
      hbObj["custom_mode"] = hb.custom_mode;
    }
    
    mavlink_global_position_int_t pos;
    if (mavlink.getPosition(pos)) {
      JsonObject posObj = payload.createNestedObject("position");
      posObj["lat"] = pos.lat / 1e7;  // Convertir a grados decimales
      posObj["lon"] = pos.lon / 1e7;
      posObj["alt"] = pos.alt / 1000.0; // mm → m
      posObj["hdg"] = pos.hdg / 100.0;  // centideg → grados
    }
    
    // Enviar
    gsm.sendTelemetry(payload);
  }
  
  // 3. Yield/background tasks
  delay(0); // Opcional: permite background tasks WiFi/GSM
}
```

## 5. Implementación Detallada por Clase

### 5.1 GsmManager
**Archivos:** `libs/GsmManager.h`, `libs/GsmManager.cpp`

**Constructores:**
```cpp
GsmManager(HardwareSerial& serial);
```

**AT Commands Clave:**
- `AT+CPIN?` → Verificar SIM
- `AT+CREG?` → Verificar registro red
- `AT+CGATT?` → Verificar GPRS attach
- `AT+SAPBR=3,1,"CONTYPE","GPRS"` → Configurar GPRS
- `AT+SAPBR=3,1,"APN","su-apn"` → APN
- `AT+SAPBR=1,1` → Abrir GPRS
- `AT+FSLS=/cert/` → Listar certificados
- `AT+CFSWRITE="filepath",0,size` → Escribir certificado
- `AT+MQTTCONF=` → Configurar MQTT (SSL, keepalive, QoS)
- `AT+MQTTSTART` → Iniciar cliente
- `AT+MQTTCONN` → Conectar a AWS endpoint (puerto 8883)
- `AT+MQTTPUB` → Publicar payload

**Reconexión agresiva:**
- `lastGprsAttempt`: si falla, reintentar cada 5000ms
- `gprsFailCount`: si ≥ 3 → `modem.restart()` y reset count
- MQTT: similar pero independiente

**Envío:**
- Construir JSON con ArduinoJson en main.ino
- Serializar a string → `AT+MQTTPUB="topic","payload"`

### 5.2 MavlinkHandler
**Archivos:** `libs/MavlinkHandler.h`, `libs/MavlinkHandler.cpp`

**Configuración:**
- Serial2.begin(57600)
- mavlink_set_protocol_version(&chan, MAVLINK_V2)

**Update():**
```cpp
void MavlinkHandler::update() {
  while (Serial2.available()) {
    uint8_t c = Serial2.read();
    mavlink_message_t msg;
    if (mavlink_parse_char(MAVLINK_COMM_0, c, &msg, &status)) {
      switch (msg.msgid) {
        case MAVLINK_MSG_ID_HEARTBEAT:
          mavlink_msg_heartbeat_decode(&msg, &lastHeartbeat);
          lastHeartbeatMs = millis();
          break;
        case MAVLINK_MSG_ID_GLOBAL_POSITION_INT:
          mavlink_msg_global_position_int_decode(&msg, &lastPosition);
          hasValidPosition = true;
          break;
      }
    }
  }
}
```

**Timeouts:**
- `isConnected()`: `(millis() - lastHeartbeatMs) < 5000`

### 5.3 SensorManager
**Archivos:** `libs/SensorManager.h`, `libs/SensorManager.cpp`

**ADC Config:**
```cpp
analogReadResolution(12); // ESP32 12-bit por defecto
analogSetAttenuation(ADC_11db); // Rango 0-3.3V
```

**MAX31865 (SPI):**
- Modo: PT100, 3-wire (ajustable)
- Leerregistro de temperatura → ohms → °C (fórmula PT100 Callendar-Van Dusen)
- Librería: Adafruit_MAX31865 (verificar disponibilidad en PlatformIO/Arduino IDE)

**Filtrado:**
```cpp
// Promedio móvil 4 muestras
float SensorManager::applyFilter(float newValue, float* buffer) {
  buffer[bufferIdx] = newValue;
  bufferIdx = (bufferIdx + 1) % 4;
  return (buffer[0] + buffer[1] + buffer[2] + buffer[3]) / 4.0;
}
```

## 7. Estrategia de Reconexión (Agresiva)

| Evento | Acción | Retry Interval |
|--------|--------|----------------|
| GPRS attach falla | Reintentar `AT+CGATT?` | 5s |
| GPRS activación falla | Reset `SAPBR` y reintentar | 5s |
| MQTT connect falla | Reintentar conexión | 5s |
| MQTT publish falla | Asume desconectado → reconectar | 5s |
| 3 fallos consecutivos GPRS | `modem.restart()` | — |
| 3 fallos consecutivos MQTT | Recrear conexión GPRS si es necesario | 5s |

**Timeouts:**
- GPRS attach: 30s max por intento
- MQTT connect: 30s max por intento
- MQTT publish: 10s

## 13. Suposiciones y Aclaraciones

- **Certificados**: El usuario proporcionará archivos `.h` con `R"EOF(...)EOF"` en `certs/`. El plan asume que las cadenas están completas y en el formato correcto PEM.
- **APN del operador**: Se debe configurar `APN` en GsmManager (const `DEFAULT_APN`). El usuario deberá modificarlo según su SIM (ej. `"internet"`, `"movistar.pe"`, etc.).
- **AWS IoT endpoint**: Se pasa como parámetro a `gsm.begin()`. Ejemplo: `"a3k7odshaiipe8-ats.iot.us-east-1.amazonaws.com"`
- **MQTT topic**: Se asume `usv/telemetry` para envío, `usv/command` para recepción. Cambiable en código.
- **Formato MAVLink**: Solo se procesan dos mensajes (HEARTBEAT y GLOBAL_POSITION_INT). Otros mensajes son ignorados.
- **MAX31865**: Se asume configuración PT100 3-hilos. Si es termopar, ajustar constructor.
- **Calibración sensores**: Fórmulas de conversión (voltaje→valorse físicos) deben ser proporcionadas por el usuario o calibrarse en campo. El código incluirá placeholders con comentarios.
- **Fuente de alimentación**: Se asume que el ESP32 y A7670SA tienen fuente estable (5V/2A mínimo).
